# 需求文档 - Tri-Transformer 前端

## 1. 业务目标

为 **Tri-Transformer 可控对话与 RAG 知识库增强系统** 构建可视化 Web 前端，提供：
- 高可用的多轮对话界面，支持知识来源追溯
- RAG 知识库管理面板
- 模型训练监控与性能指标可视化仪表盘

**目标用户**：技术开发者、领域专家、家庭管理者

**成功指标**：
- 用户可通过 Web 界面完成多轮对话并看到知识来源引用
- 文档上传后可直接基于文档内容提问
- 管理员可通过面板管理知识库与监控模型训练状态

## 2. 功能范围

### 在范围内
- 对话界面（单轮/多轮对话、知识来源追溯、历史导出）
- 文档上传与提问功能
- RAG 知识库管理面板（上传、删除、检索测试）
- 模型训练状态监控界面
- 性能指标可视化（检索准确率、BLEU 分数、幻觉率）
- 用户身份认证界面（登录/注册）

### 不在范围内
- 后端 API 实现
- Tri-Transformer 模型实现
- RAG 引擎实现
- 移动端 APP

## 3. 技术栈

| 层级 | 选型 |
|------|------|
| 框架 | React 18 + TypeScript |

| 状态管理 | Zustand |
| 构建工具 | Vite |
| 测试 | Vitest + React Testing Library |

## 4. 关键交互路径

### 路径 1：对话流程
用户登录 → 进入对话界面 → 输入问题 → 查看回复与知识来源引用

**状态流转**：空闲 → 生成中（Loading）→ 完成 / 错误

### 路径 2：文档上传与提问
上传文档（PDF/Word/MD）→ 系统处理 → 文档变为可用 → 基于文档提问

**状态流转**：上传中 → 处理中 → 可用 / 失败

### 路径 3：知识库管理
管理员登录 → 进入知识库管理 → 上传/删除文档 → 测试检索效果

### 路径 4：监控面板
查看实时/历史的检索准确率、BLEU 分数、幻觉率趋势图
