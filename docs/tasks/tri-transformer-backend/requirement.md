# Tri-Transformer 后端服务 - 需求文档

**任务 ID**: tri-transformer-backend
**平台**: Python / FastAPI
**创建时间**: 2026-03-27

---

## 一、需求范围

本文档描述 Tri-Transformer 可控对话与 RAG 知识库增强系统**后端部分**的开发需求。

**包含范围：**
- FastAPI 后端服务框架搭建
- RAG 知识库引擎（文档摄入、分块、向量化、检索、重排）
- 对话管理 API（会话、多轮对话、历史记录）
- 知识库管理 API（文档 CRUD、向量库操作）
- Tri-Transformer 模型推理接口（含 Mock 模式）
- 用户认证与权限管理（JWT）
- 训练任务调度 API
- 结果后处理（事实校验、格式优化）

**不包含范围：**
- 前端 React 界面（单独任务处理）
- 模型训练本身（提供接口调用）

---

## 二、核心功能需求

### FR-001: 多格式文档摄入
- 支持 PDF、DOCX、MD 格式上传
- 上传后自动触发分块与向量化
- 返回文档 ID 与处理状态

### FR-002: 文档分块与 BGE 向量化
- 支持固定窗口分块（chunk_size 可配置）
- 使用 BGE-large-zh-v1.5 生成向量
- 向量与元数据存入 Chroma（个人）/ Milvus（企业）

### FR-003: 混合检索与重排
- 向量检索 + BM25 混合检索
- BGE Reranker 重排精准度提升
- 元数据过滤（kb_id、时间、类型）
- 响应时间 < 500ms

### FR-004: 多轮对话 API
- 会话创建与管理
- 消息发送（触发 RAG + 推理完整链路）
- 对话历史查询
- 知识来源引用返回

### FR-005: 知识库管理 API
- 文档上传/列表/删除
- 多租户（kb_id）隔离
- 检索测试接口

### FR-006: Tri-Transformer 推理接口
- 三分支推理调用（I/C/O Transformer）
- 支持 Mock 模式（无 GPU 环境）
- 推理失败自动重试

### FR-007: 用户认证（JWT）
- 注册/登录
- Bearer token 保护接口
- 多租户数据隔离

### FR-008: 训练调度 API（P1）
- 提交/查询/取消训练任务
- 异步执行，状态追踪

### FR-009: 事实一致性后处理（P1）
- 知识来源引用标注
- 不一致内容降低置信度

## 三、非功能需求

| 指标 | 要求 |
|------|------|
| 检索响应时间 | < 500ms（Top 10） |
| 生成时间 | < 2s（1024 token） |

| 可用性 | > 99.9% |
| 安全 | JWT 认证，数据加密 |

## 四、技术栈

| 组件 | 选型 |
|------|------|
| 语言/框架 | Python 3.10+ / FastAPI |
| RAG 引擎 | LlamaIndex |
| 向量数据库 | Chroma（MVP）|
| 嵌入模型 | BGE-large-zh-v1.5 |
| 重排模型 | BGE-Reranker-Large |
| 元数据库 | PostgreSQL |
| 文档存储 | MinIO / 本地文件系统 |
| 认证 | JWT（python-jose）|
| 测试 | pytest + pytest-asyncio |
